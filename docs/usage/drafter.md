# Drafter Agent — User Guide

## What it is

The Drafter Agent generates petition drafts grounded in the jurisprudence repertory. Every citation is verified against real sources — no hallucinated references reach the final document.

## What it is NOT

- Not a replacement for the lawyer's judgment — drafts require review and editing
- Not a template filler — it generates contextual arguments based on case data and jurisprudence
- Not a final document — expect 30min of editing before filing

## Usage

```bash
# Basic draft (local LLM)
uv run juris draft <numero_cnj> contestacao

# With specific thesis
uv run juris draft <numero_cnj> contestacao --thesis "Prescricao quinquenal aplica-se ao caso"

# Using cloud LLM (Claude)
uv run juris draft <numero_cnj> contestacao --cloud

# Save to file
uv run juris draft <numero_cnj> contestacao --output ./drafts/

# Skip post-generation review
uv run juris draft <numero_cnj> contestacao --skip-review

# Custom instructions
uv run juris draft <numero_cnj> inicial --instructions "Enfatizar dano moral in re ipsa"
```

## Options

| Flag | Description | Default |
|------|-------------|---------|
| `--thesis`, `-T` | Explicit thesis statement | Auto-inferred |
| `--cloud` | Use Claude instead of local Ollama | `False` |
| `--output`, `-o` | Save draft files to directory | Print to stdout |
| `--skip-review` | Skip ReviewerAgent post-check | `False` |
| `--instructions`, `-i` | Custom instructions for the LLM | `""` |
| `--tribunal`, `-t` | Tribunal ID | `tjmg` |

## Petition Types

`inicial`, `contestacao`, `apelacao`, `agravo_instrumento`, `embargos_declaracao`, `recurso_especial`, `recurso_extraordinario`, `contrarrazoes`, `cumprimento_sentenca`, `execucao`

## Output

The draft includes:
1. **Petition body** with resolved citations (human-readable labels)
2. **[CONTRAPONTO PREVISTO]** section — strongest opposing arguments found in jurisprudence
3. **Summary footer** — research coverage, citation count, revisions performed, reviewer findings

When `--output` is used, two files are saved:
- `draft.md` — the petition body
- `draft.contraponto.md` — the contraponto section separately

## Citation Contract

The drafter uses `[CITE:source_id]` markers during generation. These are:
1. Verified against the repertory (deterministic, no LLM)
2. Resolved to human-readable labels in the final output (e.g., "Sumula 297 do STJ")
3. Failed markers trigger a re-prompt (up to 1 revision round by default)

This ensures **zero hallucinated citations** in the output.

## Contraponto Section

Every draft includes a `[CONTRAPONTO PREVISTO]` section that:
- Identifies the strongest opposing jurisprudence found
- Shows the hierarchy level and tribunal of each opposing source
- Helps the lawyer prepare for counterarguments

This is generated by the Researcher's antithesis loop — real jurisprudence, not LLM fabrication.

## Audit Trail

Every draft generates a full audit chain:
- `draft.context_built` — case context assembled
- `draft.thesis_chosen` — thesis selected or inferred
- `research` — supporting and opposing jurisprudence found
- `llm_call` — each LLM generation call
- `draft.citations_verified` — verification results
- `draft.reviewed` — reviewer findings (if not skipped)
- `draft.completed` — final draft with metadata
